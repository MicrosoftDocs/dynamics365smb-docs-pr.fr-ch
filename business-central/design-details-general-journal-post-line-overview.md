## <a name="old-architecture"></a><span data-ttu-id="74616-110">Ancienne architecture</span><span class="sxs-lookup"><span data-stu-id="74616-110">Old Architecture</span></span>  
<span data-ttu-id="74616-111">L'ancienne architecture avait les fonctions suivantes :</span><span class="sxs-lookup"><span data-stu-id="74616-111">The old architecture had the following features:</span></span>  
  
* <span data-ttu-id="74616-112">Il y a eu une utilisation extensive des variables globales, qui ont augmenté la possibilité d'erreurs masquées dues à l'utilisation de variables avec une portée incorrecte.</span><span class="sxs-lookup"><span data-stu-id="74616-112">There was extensive use of global variables, which increased the possibility of hidden errors due to use of variables with the wrong scope.</span></span>  
* <span data-ttu-id="74616-113">Il y a eu beaucoup de procédures longues (avec plus de 100 lignes de code) qui présentaient également une complexité cyclomatique élevée (c'est-à-dire, beaucoup de formulations imbriquées CASSE, RÉPÉTITION, SI), ce qui a rendu le code très difficile à mire et modifier.</span><span class="sxs-lookup"><span data-stu-id="74616-113">There were many long procedures (with more than 100 code lines) that also had high cyclomatic complexity (that is, a lot of CASE, REPEAT, IF nested statements), which made the code very difficult to read and maintain.</span></span>  
* <span data-ttu-id="74616-114">Plusieurs procédures qui n'étaient utilisées que localement et ne devaient être utilisées que localement n'ont pas été marquées comme locales.</span><span class="sxs-lookup"><span data-stu-id="74616-114">Several procedures that were only used locally and were only meant to be used locally were not marked as local.</span></span>  
* <span data-ttu-id="74616-115">La plupart des procédures n'avaient pas de paramètres et utilisaient des variables globales.</span><span class="sxs-lookup"><span data-stu-id="74616-115">Most procedures had no parameters and used global variables.</span></span> <span data-ttu-id="74616-116">Certains utilisaient des paramètres et écrasaient les variables globales par des valeurs locales.</span><span class="sxs-lookup"><span data-stu-id="74616-116">Some used parameters and overrode global variables with locals.</span></span>  
* <span data-ttu-id="74616-117">Les combinaisons de code pour trouver les comptes généraux et créer des écritures comptables et les écritures TVA n'ont pas été standardisées et n'ont pas été variées d'un endroit à l'autre.</span><span class="sxs-lookup"><span data-stu-id="74616-117">Code patterns for searching the general ledger accounts and creating general ledger and VAT entries was not standardized and varied from place to place.</span></span> <span data-ttu-id="74616-118">En outre, il y a beaucoup de duplication de code et une symétrie rompue entre le code client et fournisseur.</span><span class="sxs-lookup"><span data-stu-id="74616-118">In addition, there was a lot of code duplication and broken symmetry between customer and vendor code.</span></span>  
* <span data-ttu-id="74616-119">Une grande partie du code dans le codeunit 12, approximativement 30 %, est lié aux calculs d'escompte et écart, même si ces fonctions ne sont pas nécessaires dans de nombreux pays ou régions.</span><span class="sxs-lookup"><span data-stu-id="74616-119">A large part of the code in codeunit 12, approximately 30 percent, related to payment discount and tolerance calculations, although these features are not needed in many countries or regions.</span></span>  
* <span data-ttu-id="74616-120">Validation, Appliquer, Désappliquer, Contrepasser, Escompte et écart de paiement, et Ajustement du taux de change ont été associés dans le odeunit 12 à l'aide d'une longue liste de variables globales.</span><span class="sxs-lookup"><span data-stu-id="74616-120">Posting, Apply, Unapply, Reverse, Payment Discount and Tolerance, and Exchange Rate Adjustment were married together in codeunit 12 using a long list of global variables.</span></span>  
  
### <a name="new-architecture"></a><span data-ttu-id="74616-121">Nouvelle architecture</span><span class="sxs-lookup"><span data-stu-id="74616-121">New Architecture</span></span>  
<span data-ttu-id="74616-122">Dans [!INCLUDE[d365fin](includes/d365fin_md.md)], le codeunit 12 présente les améliorations suivantes :</span><span class="sxs-lookup"><span data-stu-id="74616-122">In [!INCLUDE[d365fin](includes/d365fin_md.md)], codeunit 12 has had the following improvements:</span></span>  
  
* <span data-ttu-id="74616-123">Le Codeunit 12 a été remanié en procédures plus petites (toutes inférieures à 100 lignes de code).</span><span class="sxs-lookup"><span data-stu-id="74616-123">Codeunit 12 has been refactored into smaller procedures (all less than 100 code lines).</span></span>  
* <span data-ttu-id="74616-124">Des motifs standardisés pour la recherche des comptes généraux ont été mis en œuvre à l'aide de les fonctions de participation des tableaux de groupes de validation.</span><span class="sxs-lookup"><span data-stu-id="74616-124">Standardized patterns for the search of general ledger accounts have been implemented by using helper functions from Posting Group tables.</span></span>  
* <span data-ttu-id="74616-125">Un cadre de moteur de validation a été mis en œuvre pour gérer le début et la fin des transactions et pour trouver la création dans les écritures comptables et les écritures TVA, la collection d'ajustement TVA, et le calcul des montants supplémentaires en devise.</span><span class="sxs-lookup"><span data-stu-id="74616-125">A Posting Engine Framework has been implemented to manage the start and finish of transactions and to isolate the creation to general ledger and VAT entries, the collection of VAT adjustment, and the calculation of additional currency amounts.</span></span>  
* <span data-ttu-id="74616-126">La duplication de code a été éliminée.</span><span class="sxs-lookup"><span data-stu-id="74616-126">Code duplication has been eliminated.</span></span>  
* <span data-ttu-id="74616-127">De nombreuses fonctions de participation ont été transférées vers les tableaux d'écritures comptables client et fournisseur correspondantes.</span><span class="sxs-lookup"><span data-stu-id="74616-127">Many helper functions have been transferred to corresponding customer and vendor ledger entry tables.</span></span>  
* <span data-ttu-id="74616-128">L'utilisation des variables globales a été réduite, de façon à ce que chaque procédure utilise des paramètres et contienne sa propre logique d'application.</span><span class="sxs-lookup"><span data-stu-id="74616-128">The use of global variables has been minimized, so that each procedure uses parameters and encapsulates its own application logic.</span></span>
